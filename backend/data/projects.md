## GUITestBench：面向移动应用缺陷发现的 GUI Agent 评测基准

- **类型**：会议论文
- **状态**：NeurIPS 2026 在投，署名作者
- **导师**：桑基韬教授，北京交通大学
- **定位**：面向移动应用真实缺陷发现任务的 GUI Agent 评测基准，关注 Agent 是否能自主复现 GitHub Issue、定位异常行为并完成可验证的缺陷发现。
- **数据规模**：覆盖 18 款真实开源 App、143 个 GitHub Issue 驱动的缺陷复现任务，任务包含缺陷规约、运行环境、视觉状态与行为轨迹等多源信息。
- **本人贡献**：参与设计 LLM-as-Judge 自动裁判框架，定义融合缺陷规约、截图与行为轨迹的多源判定方法，构建 Defect Recall、Defect Precision、Defect F1 等指标体系，并参与 UI-TARS、GUI-Owl 等主流 GUI Agent 的统一基准评测、对比分析与消融实验。
- **能力体现**：GUI Agent 评测、LLM-as-Judge、实验设计、指标体系构建、多模态轨迹分析、Python 实验管线。
- **关键词**：GUI Agent、移动应用测试、缺陷发现、LLM-as-Judge、Benchmark、UI-TARS、GUI-Owl、NeurIPS 2026

## 基于多模态智能体协同推理的可解释视频取证框架

- **类型**：科研项目
- **资助**：北京市自然科学基金本科生"启研"计划，资助 5 万元
- **时间**：2026 年至今
- **导师**：赵耀教授（IEEE Fellow），依托科幻音视频智能处理北京市重点实验室
- **定位**：面向 AI 生成、篡改或伪造视频的可解释取证框架，希望让模型不只给出真假判断，还能说明证据来自哪里、为什么可信。
- **团队角色**：本人作为核心成员参与项目推进，团队共 2 人。
- **本人贡献**：负责设计基于 MCP 协议的异构工具链协同调度机制，研究 MoE 架构下物理规律、语义自洽与空时一致性三维度专家的正交化训练策略与跨模态证据融合方法，参与构建基于思维链诱导的结构化 n-元组可解释取证输出协议。
- **研究特色**：将多模态智能体、工具调用、专家模型、跨模态证据融合和可解释输出结合起来，让视频取证从单一分类走向“证据链式推理”。
- **预期成果**：CCF-A/B 论文 1–2 篇
- **关键词**：多模态智能体、MCP 协议、MoE、视频取证、可解释 AI、跨模态证据融合、Chain-of-Thought

## 基于 RISC-V 架构的 Rust 轻量级实时操作系统设计与实现

- **类型**：科研项目（大学生创新训练项目）
- **状态**：已通过结题，最终评级待公布（校级以上）
- **导师**：高勃教授，北京交通大学
- **定位**：在 RISC-V 64 位架构上使用 Rust 从零构建轻量级实时操作系统内核，验证 Rust 所有权机制在嵌入式实时系统中的工程价值。
- **系统能力**：实现抢占式多任务调度、基于原子指令的同步原语、8 项系统调用，以及 UART、VGA、PCI 等多设备驱动。
- **本人贡献**：负责编译器工具链适配与文件系统模块开发，参与内核模块联调，并通过将 unsafe 代码约束至最小边界，提高系统内存安全性与可维护性。
- **能力体现**：Rust、RISC-V、操作系统内核、文件系统、编译工具链、底层驱动、并发同步、内存安全。
- **关键词**：Rust、RISC-V、RTOS、操作系统、内存安全、文件系统、编译器工具链、嵌入式系统
